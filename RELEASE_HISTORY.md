# 배포 이력

이 문서는 프로젝트의 모든 배포 이력을 기록합니다.

## 배포 이력 기록 형식

```markdown
## [버전] - YYYY-MM-DD

### 배포 정보
- **배포 날짜**: YYYY-MM-DD
- **배포 담당자**: [이름]
- **배포 방법**: 수동 / 자동
- **빌드 번호**: +X

### 변경 사항
- [변경 내용]

### 문제 발생 여부
- 없음 / [문제 설명]

### 롤백 여부
- 없음 / 롤백됨 (vX.X.X로 롤백)
```

## 배포 이력 작성 가이드

### 배포 시 기록할 항목

1. **배포 정보**
   - 배포 날짜
   - 배포 담당자
   - 배포 방법 (수동/자동)
   - 빌드 번호

2. **변경 사항**
   - CHANGELOG.md의 해당 버전 내용 참고
   - 주요 변경사항 요약

3. **문제 발생 여부**
   - 배포 후 발견된 문제
   - 사용자 피드백
   - 크래시 리포트

4. **롤백 여부**
   - 롤백이 발생한 경우
   - 롤백 사유
   - 롤백 후 버전

### 배포 후 업데이트

배포 후 다음 정보를 업데이트합니다:

- 실제 배포 날짜
- 배포 후 문제 발생 여부
- 사용자 피드백
- 성능 지표 (있는 경우)
